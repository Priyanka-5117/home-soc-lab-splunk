4. Technical Analysis
Splunk SIEM detected 34 instances of PowerShell execution via
Windows Security EventCode 4688 (Process Creation). PowerShell
is commonly used by attackers for:
- Downloading and executing malicious payloads (T1059.001)
- Bypassing security controls using encoded commands
- Establishing C2 communication channels
- Performing reconnaissance and lateral movement

**Key Observations:**
- 34 PowerShell executions detected in short timeframe
- Process created by SOCUser — admin account
- New_Process_Name contained powershell.exe
- High frequency of executions indicates scripted activity
- No malicious payload confirmed in this instance

10. Lessons Learned
PowerShell is one of the most abused tools in modern cyberattacks.
The ability to detect and investigate PowerShell execution in
real-time using Splunk and Windows EventCode 4688 is a critical
SOC capability. This incident highlights the need for deeper
PowerShell logging including Script Block Logging (EventCode 4104)
to capture the actual commands being executed, not just the
process creation event.
