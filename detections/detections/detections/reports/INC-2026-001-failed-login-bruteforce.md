Incident Report — INC-2026-001
**Brute Force Login Attack | Credential Access | MITRE T1110**


1. Executive Summary
A brute force credential attack was detected on Windows endpoint 
DESKTOP-HEC7FEF within the SOC home lab environment. The Splunk 
SIEM triggered an alert after identifying 10 consecutive failed 
authentication attempts against a local account within a 1-second 
window — a pattern consistent with automated password guessing tools.
The incident was contained within the lab environment and no 
unauthorized access was achieved.

 4. Technical Analysis
The Splunk SIEM detected 10 failed Windows Security Event 4625 
(Failed Logon) entries originating from localhost (::1) targeting 
the account "fakeuser" within a single second. This velocity of 
failed attempts is characteristic of automated brute force tools 
such as Hydra or Medusa.

**Key Observations:**
- All 10 attempts occurred within 1 second — indicating automation
- Source IP was ::1 (localhost) — suggesting local execution
- Target account "fakeuser" does not exist — indicating enumeration
- No successful logon (EventCode 4624) followed the attempts
- Windows Logon Type 2 (Interactive) was used

 7. Detection Rule (SPL)
```spl
index=windows sourcetype="WinEventLog:Security" EventCode=4625
| stats count by Account_Name, Source_Network_Address
| where count > 3
| eval mitre_technique="T1110 - Brute Force"
| eval severity="High"
| table Account_Name, Source_Network_Address, count, mitre_technique, severity
```

10. Lessons Learned
This incident highlights the importance of velocity-based detection 
rules. Traditional signature-based detection may miss low-and-slow 
brute force attacks. The Splunk SPL rule successfully identified the 
attack pattern within seconds of occurrence, demonstrating the value 
of SIEM-based behavioral detection mapped to MITRE ATT&CK framework.
